Модуль миграций.

1.Инициализация.
	1.1. Создаются служебные директории и фаилы модуля:
		./migrations/ - главная директория модуля;
		./migrations/magic/ - директория с версиями миграций;
		./migrations/history.db - база данных для хранения истории миграций;
	1.2. Инициализируется объект управления миграциями.
		Режим development:
	
	1.2.1. Инициализируется объект миграции:
			Создается соединение со служебной базой данных, формируется модель orma.
	1.2.2. Загружаетя последняя версия миграции.
			Проверка наличия ошибок. 
				-да-> Вывод ошибок в консоль. Выход.
	1.2.3. Выделяется информация о различиях схемы app и последней миграции.
			Вывод информации в консоль.
	
		Режим migration:

	1.2.1. Инициализируется объект миграции:
			Создается соединение со служебной базой данных, формируется модель orma.
	1.2.2. Загружаетя последняя версия миграции.
			Проверка наличия ошибок. 
				-да-> Вывод ошибок в консоль. Выход.
			Проверка на первую миграцию 
				-да-> Выход.
			Проверка на завершенность последней миграции.
			Проверяется существование текущей маджик функции.
			 	-да-> Выход.
			Создание фаила magic. Вывод в консоль пути к фаилу magic.
		
2. Миграция, только режим migrаtion (id)	
	2.1. Загружается последняя версия миграции
			Проверка наличия ошибок. 
				-да-> Вывод ошибок в консоль. Выход.
	2.2. Проверка аргумента Id
			-да-> загружается схема соответствующей миграции.
			-нет-> загружается схема app
	2.3. Выделяется информация о различиях текущей схемы и последней миграции.
			Загружается magic из фаила.
			Проверка на завершенность последней миграции.
				-нет-> синхронизируется последняя миграция.
				-да-> Синхронизируется новый объект миграции.
	2.4. Проверка есть ли новые аттрибуты или модели
			-да- Выполняется ddl-sync текущей схемы расширенной новыми аттрибутами и моделями
	2.5. Reload scope
	2.6. Выполняется orm-sync для всех баз данных
	2.7. Проверка на удаление аттрибутов.
			-да - ddl-sync схемы без удаленных аттрибутов
	2.8. Удаление моделей.
		  
3. Информация, только режим migration (id)
	3.1. Загружаетя последняя версия миграции.
			Проверка наличия ошибок. 
				-да-> Вывод ошибок в консоль. Выход.
	3.2. Проверка аргумента Id
			-да-> загружается схема соответствующей миграции.
			-нет-> загружается схема app
	3.3 Выделяется информация о различиях схемы и последней миграции.
			Вывод информации в консоль.  
			
4. Sync, только режим migration (id, force)
	2.0. При наличии флага force игнорируется всё, что могло помешать миграции.
	2.1. Загружается последняя версия миграции
			Проверка наличия ошибок. Вывод ошибок в консоль.
	2.2. Проверка аргумента Id
			-да-> загружается схема соответствующей миграции.
			-нет-> загружается схема app
	2.3. Выделяется информация о различиях текущей схемы и последней миграции.
			-да- Выполняется ddl-sync новой схемы
	2.6. Выполняется orm-sync для всех баз данных
